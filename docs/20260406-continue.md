## Recommended Next Steps

### 1. Exclude asm + alloca from in-scope (immediate)

Update test harness and counts. Reduces denominator to 34.

### 2. GNU `?:` ternary extension (arith test)

The Elvis operator `a ?: b` means `a ? a : b`. Small parser change
in `parse_ternary`: if `:` follows `?` immediately, reuse the
condition as the then-expression. Unlocks the arith test.

### 3. `__attribute__` syntax (attribute test)

Accept and skip `__attribute__((...))` after function/variable
declarations. Just consume balanced parens. Unlocks attribute test.

### 4. Improve awk filter (bitfield, initializer, struct, typedef)

The `/short/` filter strips ALL lines containing "short", which
removes struct members like `short a;`. Solutions:
- Change to strip only declaration-level `short` (not struct members)
- Or use per-test override filters
- Could unlock 3-4 tests

### 5. Const-fold expressions (constexpr test)

Support `+`, `-`, `*`, `/` in constant expressions (enum values,
array sizes, `#if` conditions). The `parse_const_array_size` already
handles some; extend to cover more operators.

### 6. bgc categorization

Access the Linux machine to categorize the 117 bgc examples:
- Count how many use float/double (exclude from in-scope)
- Re-run compilation with latest compiler
- Update counts

## Architecture Notes

### Statement Expression Scoping

The current approach: `collect_locals` and `gen_expr` both save/restore
the locals map (names, types, locals_size) around StmtExpr blocks.
`gen_local_decl` allocates on-demand for variables missing from the
restored map. `locals_size` is reset to 0 after the prologue so
on-demand allocation matches the collect pass.

This works correctly for all current tests but is fragile — both
passes must mirror each other exactly. A more robust approach would
be to rename variables at parse time (e.g., `x$scope_id`) to avoid
map collisions entirely.

### Codegen Optimization Opportunities (not yet done)

- **Register allocation**: Keep hot variables in r2 (callee-saved).
  Would close the remaining 1.5x gap to hand-optimized code.
- **Constant folding**: Evaluate `2+3` at compile time.
- **Dead store elimination**: Skip zero-fill for explicitly initialized elements.
