Fix two pre-existing codegen storage bugs surfaced during the string-literal-concatenation saga: (1) char *g = "abc" stores string bytes at the symbol address instead of allocating an anonymous label for the bytes and storing the address in _g; (2) char a[] = "abc" local reserves len bytes on stack but stores a pointer to rodata in the first slot instead of copying the bytes in. Both are symmetric init/access mismatches. Brief: /disk1/github/softwarewrighter/devgroup/tools/briefs/dcxtc-codegen-string-storage-bugs.md
