Extend tc24r's array-size grammar to accept compile-time integer constant expressions inside [ ... ]. Currently only a single literal or #define-resolved identifier is accepted (e.g. [BUF_SIZE] works, [A*B] fails with 'expected RBracket, got Star'). Support binary * / + - with standard C precedence (left-associative, * / higher than + -), unary -, and parens for grouping. Identifiers are resolved through #define lookup; non-constant identifiers must error with a clear message naming the offender ('must be a compile-time integer constant'). Constant-fold to a single integer at parse time so codegen matches the literal-sized equivalent. Out of scope: sizeof, function calls, VLAs. Add positive tests (A*B, N+1, (A+B)*C, N*16, A*(B+C)) verifying codegen matches literal-sized equivalents, and negative tests (runtime variable, broken expression A*).
