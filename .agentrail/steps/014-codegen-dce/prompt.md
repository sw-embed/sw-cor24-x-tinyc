Whole-program dead-code elimination in tc24r. Today every function defined in any included .h ships in the .s output whether called or not — verified by grep on demo45's .s which contains _free, _calloc, _realloc, _exit, _abs, _atoi though only malloc is used. tc24r is single-TU so whole-program analysis is straightforward at the AST level (between parse and codegen). Build a call graph from main + interrupt handlers + globals' init expressions; do reachability traversal; filter Program.functions to the reachable set before codegen. Conservative defaults for inline asm (e.g. keep all functions if any asm appears, or scan asm strings for symbol-like tokens). Tests: direct call kept / unreferenced dropped / transitive reachability / function pointer keeps target / interrupt handler is a root / demo45 .s shrinks (no _abs etc.) / regression (existing demos still produce same r0/halt/UART).
